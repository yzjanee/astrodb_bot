# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

A collection of **Claude Code agent skills** for interactive AstroDB ingestion workflows. Each skill lives under `skills/<skill-name>/SKILL.md` and is a standalone instruction document for an AI agent. The repo is packaged as a Claude Code plugin (`.claude-plugin/plugin.json`).

There is no application code to compile or run directly — the "code" is primarily markdown skill instructions plus Python helper scripts under each skill's `scripts/` directory.

## Commands

Install dependencies (Python 3.13 required):
```bash
uv sync
```

Run a single eval for a skill (no dedicated test runner — evals are JSON files run through Claude):
```bash
# Evals are at skills/<skill-name>/evals/evals.json
# To inspect: cat skills/astrodb-setup/evals/evals.json
```

Run felis schema validation (used by multiple skills):
```bash
uv run felis validate <path-to-schema.yaml>
```

Run database tests generated by `astrodb-create-db`:
```bash
cd <user-project-root>
uv run pytest tests/ -v
```

## Architecture

### Skill Structure

Every skill follows this layout:
```
skills/<skill-name>/
├── SKILL.md          ← agent instructions (the skill itself)
├── evals/
│   └── evals.json    ← test cases with prompt + expected assertions
├── references/       ← supporting lookup docs referenced from SKILL.md
└── scripts/          ← Python helper scripts called by the skill
```

`SKILL.md` frontmatter includes `name`, `description`, and `compatibility`. The `description` is what Claude uses to decide when to trigger the skill automatically.

### Skill Pipeline (Typical Ingestion Workflow)

Skills are designed to chain in order:

1. **`astrodb-setup`** — Clone a user's repo from the astrodb-template-db GitHub template; set `db_name` in `database.toml`. No data involved.
2. **`astrodb-parse-data-table`** — Read FITS/CSV/ECSV/etc. with astropy or pandas; extract column names, descriptions, units, types. Writes a sidecar to `tmp/astrodb-parse-result.json` for downstream skills.
3. **`astrodb-match-schema`** — Map parsed columns to AstroDB template schema tables/fields. Reads `references/schema.md`, `references/column-patterns.md`, `references/photometry-filters.md`.
4. **`astrodb-validate-schema-mapping`** — Check for nullable violations and type mismatches between data and schema.yaml. Reads `references/validation-report.md`.
5. **`astrodb-generate-schema`** — Produce a Felis YAML `schema.yaml` from the mapping. Runs `felis validate` at the end. Reads `references/felis-syntax.md`.
6. **`astrodb-create-db`** — Create an empty SQLite database from the validated schema using `astrodbkit`. Runs `scripts/create_db.py` and `scripts/generate_tests.py`. Writes `data/reference/`, `data/source/`, `database.toml`.
7. **`astrodb-ingest-publication`** — Add papers to the `Publications` table using `astrodb_utils.publications.ingest_publication`. Always resolves DOI/bibcode from NASA ADS before writing; never passes bare shortnames.
8. **`astrodb-ingest-source`** — Add rows to the `Sources` table using `astrodb_utils.sources.ingest_source`. Requires publications to already exist.
9. **`astrodb-website`** — Clone the `astrodb-web` FastAPI template and configure it to browse the SQLite database.

### Key Conventions Across Skills

- **`SAVE_DB = False` by default** in all ingestion scripts. Skills always do a dry run first and require explicit user confirmation before setting `SAVE_DB = True`.
- **`tmp/` directory** is the working scratch space: sidecar JSON, generated ingestion scripts, and generated schema YAML all land here.
- **`uv run`** is preferred over bare `python` to ensure the correct virtual environment. Fall back to `pip install` + `python3` if `uv` is unavailable.
- **Database loading**: JSON-layout databases (astrodb-template-db) use `build_db_from_json(settings_file="database.toml")`; standalone `.sqlite` files use a direct connection.

### Evals Format

Each `evals/evals.json` has `skill_name` and an array of `evals`, each with:
- `prompt` — what the user says
- `expected_output` — prose description of correct behavior
- `files` — data files the eval depends on (paths relative to the skill's `evals/` dir)
- `assertions` (or `expectations`) — specific checkable claims about the response

### External Dependencies

- [`astrodb-template-db`](https://github.com/astrodbtoolkit/astrodb-template-db) — the canonical AstroDB schema and directory layout that skills target
- [`astrodb-web`](https://github.com/astrodbtoolkit/astrodb-web) — the FastAPI website template
- `astrodbkit` — Python ORM for AstroDB SQLite databases
- `astrodb_utils` — ingestion helpers (`ingest_publication`, `ingest_source`, `find_publication`, `build_db_from_json`)
- `lsst-felis` — Felis schema validation (`felis validate`)
